## Design Implications for Coordination Architecture

### From Propositional to Epistemic Coordination

McGilchrist's art-as-knowledge theory suggests coordination should be *experienced* not just executed. This means:

- **Current left-mode pattern**: Transfer data efficiently via structured schemas
- **Right-mode complement**: Preserve situational framing, emotional valence, operator intent context

**Concrete implementation**: Context tags on blackboard entries capture:
- `operator_engagement`: How was this information used? What action did it enable?
- `uncertainty_reason`: Why is there ambiguity here? What would resolve it?
- `intent_alignment`: Does this entry support or diverge from stated goals?

### Ambient Discovery vs Reactive Engagement

The presence_indicator.html deployment (C319) addresses discoverability gap by making async_prep engagement opportunities passively visible rather than waiting for active queries. This aligns with enactive principle that perception serves action — tools must be perceivable at moment-of-need.

**Next iteration opportunity**: Add real-time state visualization showing which brief cards have been viewed but not engaged — creating "perceptual affordances" that invite interaction without pressure.
